# ADR-006: Customer Secure Mode

- **Status:** Accepted
- **Date:** 2026-07-11

## Context

When NFC payments require customer PIN authentication, customers should never expose their PIN to the employee.

The product must provide a secure and privacy-focused authentication experience.

## Decision

Customer Secure Mode is a mandatory core product feature.

Whenever customer authentication is required (for example NFC PIN):

- The application enters Customer Secure Mode.
- Hide all employee information.
- Hide all merchant management screens.
- Hide order details.
- Display only:
  - payment amount
  - secure PIN entry interface
- The numeric keypad must be randomized every session.
- Return automatically to Employee Mode after authentication.

This feature is mandatory.

## Consequences

**Advantages:**

- Better customer privacy.
- Better security.
- Strong product differentiation.
- Reusable for future secure authentication flows.

**Disadvantages:**

- Additional UI implementation.
- Android only.

## Alternatives Considered

Using the default Android PIN screen.

Rejected because it exposes unnecessary employee information and provides a weaker customer experience.
