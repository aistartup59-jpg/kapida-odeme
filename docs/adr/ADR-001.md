# ADR-001: PaymentRequest and Transaction Relationship

- **Status:** Accepted
- **Date:** 2026-07-09

## Context

The platform must represent merchant payment orders and the individual payment events that settle them. A clear domain model is needed to support partial and hybrid payment flows.

## Decision

One order creates exactly one PaymentRequest.

A PaymentRequest may contain multiple Transactions.

Relationship:

- Merchant: 1 -> N PaymentRequest
- PaymentRequest: 1 -> N Transaction

## Consequences

- PaymentRequest is the single source of truth for an order's payment state.
- Transactions are append-only records of individual payment events.
- Hybrid and partial payment flows are supported by design.
- Reporting and status derivation always start from PaymentRequest.

## Alternatives Considered

- One Transaction per order: rejected because it cannot represent partial or hybrid payments.
