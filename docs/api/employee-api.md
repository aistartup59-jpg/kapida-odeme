# Employee API

## Overview

Employee-related operations are currently documented through the authentication module, with employee creation and onboarding flows exposed as auth endpoints.

## Current surface

- POST /auth/employee
- POST /auth/employee/accept-invitation
- POST /auth/employee/set-password
- POST /auth/employee/login

## Notes

This document groups the current employee-facing operations under a dedicated API reference while preserving the existing route structure.
