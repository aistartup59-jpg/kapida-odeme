# Mobile Flow Architecture

## Current state

The mobile application is still in a scaffolded state. The current app structure includes startup, authentication, and merchant dashboard placeholders.

## Approved flow direction

1. Startup checks backend connectivity.
2. The app presents the merchant or employee authentication experience.
3. Successful authentication leads to the merchant dashboard experience.
4. The dashboard is expected to support employee management and payment reporting.
5. Payment and transaction views follow the shared domain model described in this documentation.

## Notes

This documentation reflects the approved user journey and the current scaffolded implementation without introducing new mobile routes or behavior.
